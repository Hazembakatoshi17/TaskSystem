Security Model

JWT-based RBAC with department scoping

JWT Token Structure

{
  "alg": "HS256",  // HMAC-SHA256
  "typ": "JWT"
}
.
{
  "id":       "3",                 // Emp_Id
  "mobile":   "0501234567",
  "name":     "Ahmed Ali",
  "dept_id":  "1",
  "is_admin": "true",
  "role":     "Admin",            // maps to [Authorize(Roles="Admin")]
  "iss": "TaskSystem",
  "aud": "TaskSystemUsers",
  "exp": 1747123456                  // 7 days from issue
}
.
HMACSHA256(base64(header) + "." + base64(payload), secret)

👑 Admin Role

✅Create / update / delete tasks
✅Assign employees to tasks
✅Remove task assignments
✅View all dept tasks & assignments
✅Filter tasks by status / priority
✅Update any task status
⚠️Scoped to own department only
❌Cannot manage other dept's tasks

👤 Employee Role

✅View own assigned tasks
✅Update status of own tasks
✅View own task assignments
❌Cannot view other employees' tasks
❌Cannot create or delete tasks
❌Cannot assign anyone to tasks

Department Scoping Rules

RuleDescription
Task scopingAdmin sees tasks in projects where at least one project employee is in admin's dept
Assignment validationEmployee being assigned must have Dept_Id == admin.Dept_Id
Assignment removalAdmin can only remove assignments for employees in their own dept
Employee queryAdmin can only query assignments for employees in their dept
Task creationProject must have at least one employee in admin's dept

⚠️ Security Recommendations for Production

IssueRecommendation
Passwords stored in plain textUse BCrypt.Net or ASP.NET Core Identity to hash passwords
JWT secret in appsettings.jsonMove to environment variable or Azure Key Vault
Dept/Employee endpoints are openAdd [Authorize] to restrict to authenticated users
No rate limitingAdd app.UseRateLimiter() on login endpoint
No refresh tokensImplement refresh token rotation for long sessions

Deployment & Setup Guide

From zero to running API

Stack Requirements

ComponentRequirement
.NET SDK.NET 10
DatabaseOracle Database (Free edition works: FREEPDB1)
OSWindows / Linux / macOS
PortDefault: 7xxx (HTTPS) — configurable in launchSettings.json

Setup Steps

1
Clone & restore packages
dotnet restore
2
Configure Oracle connection in appsettings.json
"OracleDb": "User Id=YOUR_USER;Password=YOUR_PASS;Data Source=localhost:1521/FREEPDB1;"
3
Run EF migrations
dotnet ef migrations add InitialCreate dotnet ef database update
4
Run the API
dotnet run
5
Open Swagger UI
https://localhost:PORT/
RoutePrefix is set to empty string, so Swagger loads at the root path.

Typical First-Use Workflow

#ActionEndpoint
1Create a departmentPOST /api/department
2Create admin employee (IsAdmin: true, Dept_Id set)POST /api/employee
3Create regular employeesPOST /api/employee
4Login as admin → get JWT tokenPOST /api/auth/login
5Create a projectPOST /api/project
6Assign employees to projectPOST /api/project/{id}/assign/{empId}
7Create a task in the project (requires JWT)POST /api/task
8Assign employee to task (requires JWT + Admin)POST /api/taskassignment/assign
9Login as employee → update task statusPATCH /api/task/{id}/status
